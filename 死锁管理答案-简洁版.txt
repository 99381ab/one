二. 计算题答案（第11题）

银行家算法计算

初始数据：
- Available: (1, 6, 2, 2)
- P0: Allocation(0,0,3,2), Need(0,0,1,2)
- P1: Allocation(1,0,0,0), Need(1,6,5,0)
- P2: Allocation(1,3,5,4), Need(2,3,5,6)
- P3: Allocation(0,0,3,2), Need(0,6,5,2)
- P4: Allocation(0,0,1,4), Need(0,6,5,6)

（1）T0状态是否安全？

答：安全

安全序列：P0 → P1 → P3 → P2 → P4

详细步骤：
1. P0完成：(1,6,2,2) + (0,0,3,2) = (1,6,5,4)
2. P1完成：(1,6,5,4) + (1,0,0,0) = (2,6,5,4)
3. P3完成：(2,6,5,4) + (0,0,3,2) = (2,6,8,6)
4. P2完成：(2,6,8,6) + (1,3,5,4) = (3,9,13,10)
5. P4完成：所有进程完成

（2）P2请求Request(1,2,2,2)，能否分配？

答：不能分配

验证：
- Request ≤ Need: (1,2,2,2) ≤ (2,3,5,6) ✓
- Request ≤ Available: (1,2,2,2) ≤ (1,6,2,2) ✓

分配后状态：
- Available' = (0,4,0,0)
- 无进程的Need ≤ (0,4,0,0)
- 系统进入不安全状态

结论：拒绝分配

（3）如果立即满足P2请求，是否立即进入死锁？

答：不会立即进入死锁状态

说明：
- 不安全状态只是可能发生死锁
- 不等于立即死锁
- 只有形成循环等待时才真正死锁
